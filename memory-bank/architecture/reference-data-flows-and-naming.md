Compiled from full codebase traces during pipeline simplification planning (2026-02-25).
Use this as a reference when making changes to avoid breaking cross-file dependencies.

## 1. ID & Naming Conventions

### Post IDs
- **Format:** `W{week_number}-P{seq}` (e.g., `W12-P01`)
- **Generated by:** `claude_api.generate_weekly_plan()`
- **Unique within:** A week
- **Used in:** weekly plan JSON, blog-generation-results.json, Content Queue sheet, content-log.jsonl

### Pin IDs
- **Format:** `W{week_number}-{seq}` (e.g., `W12-01`)
- **Generated by:** `claude_api.generate_weekly_plan()`
- **Unique within:** A week
- **Used in:** weekly plan JSON, pin-generation-results.json, pin-schedule.json, Content Queue sheet, content-log.jsonl

### Blog Slugs
- **Format:** lowercase-hyphenated (e.g., `easy-weeknight-chicken-tacos`)
- **Generated by:** Claude in blog frontmatter `slug:` field
- **Used for:**
  - Filename: `data/generated/blog/{slug}.mdx`
  - Blog URL: `https://goslated.com/blog/{slug}` (site routes by filename, NOT frontmatter slug)
  - Hero image: `data/generated/pins/{slug}-hero.{ext}`
  - Deployed path: `content/blog/{slug}.mdx` on goslated.com repo
  - Deployed image: `public/assets/blog/{slug}{ext}` on goslated.com repo

### Image Filenames
- **Pin hero images:** `{pin_id}-hero.{ext}` (e.g., `W12-01-hero.png`)
- **Blog hero images:** `{slug}-hero.{ext}` (e.g., `easy-chicken-tacos-hero.jpg`)
  - Also saved as `{pin_id}-hero.{ext}` (dual-save in generate_pin_content.py)
- **Rendered pins:** `{pin_id}.png` (e.g., `W12-01.png`) — 1000x1500px
- **AI comparison heroes:** `{pin_id}-ai-hero.png` (being removed in simplification)
- **Regen images:** `{pin_id}-hero-regen.png` or `{prefix}-regen.png`

### Image IDs (for dedup tracking)
- **Stock:** `unsplash:{photo_id}` or `pexels:{photo_id}`
- **AI generated:** `ai_{md5_hash_of_prompt[:12]}`
- **Template:** empty string
- **Stored in:** content-log.jsonl `image_id` field, pin-generation-results.json `image_id` field
- **Dedup window:** 90 days (via `load_used_image_ids()`)

### Pin-to-Blog Linkage
- `source_post_id` field on pins → parent blog's `post_id`
- Used to: derive blog URL for pin link, map AI comparison images, group pins under blogs

## 5. Pipeline Step-by-Step Data Flow

### Step 1: Plan Generation (Monday 6am ET)
```
weekly-review.yml → generate_weekly_plan.py
  → claude.generate_weekly_plan() [Sonnet]
  → saves: data/weekly-plan-W{N}-{date}.json
  → sheets.write_weekly_review() → "Weekly Review" tab
  → STATUS = "pending_review" in cell B3
```

### Step 2: Plan Approval
```
User sets B3 = "approved" in Weekly Review tab
  → Apps Script onSheetEdit() detects B3 change
  → triggerGitHubWorkflow("generate-content")
  → generate-content.yml fires
```

### Step 3: Content Generation
```
generate-content.yml →
  3a. generate_blog_posts.py
    → blog_generator.generate_batch() → claude.generate_blog_post() [Sonnet] × 8-10
    → saves: data/generated/blog/{slug}.mdx (each post)
    → saves: data/blog-generation-results.json

  3b. generate_pin_content.py
    → For each of 28 pins:
      → generate_copy_batch() → claude.generate_pin_copy() [Sonnet] (batches of 6)
      → source_image() → _source_ai_image()
        → claude.generate_image_prompt() [GPT-5 Mini after Change 5]
        → image_gen_api.generate() [gpt-image-1.5]
        → saves hero: data/generated/pins/{pin_id}-hero.png
      → pin_assembler.assemble_pin() → Puppeteer renders HTML→PNG
        → saves rendered: data/generated/pins/{pin_id}.png
    → saves: data/pin-generation-results.json

  3c. publish_content_queue.py
    → gcs.upload_pin_images() → GCS public URLs
    → gcs.upload_blog_hero_images() → GCS public URLs
    → stores _drive_image_url, _drive_download_url in pin-generation-results.json
    → sheets.write_content_queue() → "Content Queue" tab with =IMAGE() formulas
    → slack notification

  3d. git commit + push (pin-generation-results.json, blog-generation-results.json)
```

### Step 4: Content Review & Regen
```
User reviews Content Queue tab:
  → Sets column J status per row: approved / rejected / regen_image / regen_copy / regen
  → Writes feedback in column L
  → Clicks "Run Regen" button (or sets N1 = "run")
  → Apps Script → triggerGitHubWorkflow("regen-content")

regen-content.yml → regen_content.py
  → sheets.read_regen_requests()
  → For each flagged item:
    → Regenerates image and/or copy with feedback
    → Re-renders pin via pin_assembler
    → Uploads to GCS, updates Sheet row
  → Updates pin-generation-results.json
  → Updates pin-schedule.json (if already created)
  → Slack notification
```

### Step 5: Deploy to Preview
```
All Content Queue rows reviewed (no pending_review, no regen_*)
  → Apps Script allContentReviewed() → triggerGitHubWorkflow("deploy-to-preview")

deploy-and-schedule.yml → blog_deployer.py
  → deploy_to_preview()
    → github.commit_multiple_posts() → blogs + hero images to goslated.com develop branch
    → Vercel auto-deploys preview
    → verify_urls() → checks each blog is live
    → sheets.write_deploy_status("pending_review", preview_url)
```

### Step 6: Promote to Production
```
User sets B4 = "approved" in Weekly Review tab
  → Apps Script → triggerGitHubWorkflow("promote-and-schedule")

promote-and-schedule.yml → blog_deployer.py
  → promote_to_production()
    → github merge develop → main
    → verify_urls() on production
    → _create_pin_schedule() → data/pin-schedule.json
    → _append_to_content_log() → data/content-log.jsonl
```

### Step 7: Daily Posting
```
daily-post-{morning|afternoon|evening}.yml (cron 3x daily)
  → post_pins.py
    → load_scheduled_pins(today_et, slot)
    → For each pin:
      → construct_utm_link()
      → pinterest.create_pin(title, description, link, image_url, board_id, alt_text)
      → Append to content-log.jsonl with pinterest_pin_id
```

### Step 8: Analytics & Review
```
weekly-review.yml (Monday 6am) →
  → pull_analytics.py → Pinterest Analytics API → updates content-log.jsonl
  → weekly_analysis.py → claude.analyze_weekly_performance() [Sonnet]
  → generate_weekly_plan.py (next week's plan)

monthly-review.yml (1st Monday) →
  → monthly_review.py → claude.run_monthly_review() [Opus]
```

## 7. Environment Variables

### Required for all workflows
- `ANTHROPIC_API_KEY` — Claude API
- `OPENAI_API_KEY` — Image generation (gpt-image-1.5) + image prompts (gpt-5-mini)
- `GOOGLE_SHEETS_CREDENTIALS_JSON` — Sheets + GCS authentication
- `GOOGLE_SHEET_ID` — Target spreadsheet
- `GCS_BUCKET_NAME` — Google Cloud Storage bucket
- `PINTEREST_ACCESS_TOKEN` — Pinterest API
- `PINTEREST_ENVIRONMENT` — "production" or "sandbox"
- `SLACK_WEBHOOK_URL` — Notifications
- `GITHUB_TOKEN` — Git operations on goslated.com repo
- `GOSLATED_REPO` — Target repo for blog deployment

### Optional
- `IMAGE_GEN_PROVIDER` — "openai" (default) or "replicate"
- `REPLICATE_API_TOKEN` — Only if using Flux Pro
